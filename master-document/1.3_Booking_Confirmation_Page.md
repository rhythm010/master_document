# 1.3 Booking Confirmation Page

## Scope
- Single page shown to the **client** after a booking is confirmed.
- Displays upcoming booking information (no companion details initially).
- Two-part flow: **Part 1** appears immediately after booking; **Part 2** appears X hours (currently 5 hours) before the booking time with companion details.

### 1.3.1 Overview
*   After the booking is confirmed (see 1.2.1.3), the client is redirected to the **Booking Confirmation Page**.
*   This is a **client-only** page. Companions do not have this view.
*   The page remains active until the booking is either started or cancelled.

### 1.3.2 Part 1 — Immediate Confirmation

#### 1.3.2.1 Trigger
*   Displayed immediately after the booking is confirmed.

#### 1.3.2.2 API
*   **[GET] /bookings/{id}/details** — fetches core booking data.

#### 1.3.2.3 Visible Data
*   Booking ID
*   Venue Name
*   Date & Time
*   **No companion details** are shown at this stage.

#### 1.3.2.4 Available Actions
*   **Cancel Booking** — Client can cancel the booking. On tapping cancel, a review/confirmation page is shown before the cancellation is finalized. Post-cancellation, the client is redirected to the **home screen**.

### 1.3.3 Part 2 — Pre-Booking Reveal (T−X Hours, currently 5 hours)

#### 1.3.3.1 Trigger
*   At **X hours** (configurable, currently set to **5 hours**) before the booking start time, the page updates with companion details.

#### 1.3.3.2 Additional Data Revealed
*   For **both companions** (Captain and Vice Captain):
    *   **Display Name** (set during onboarding, never real name).
    *   **Ratings**.
    *   **Languages spoken**.
    *   **Sketch profile picture**.
*   All previously shown booking info (Booking ID, Venue, Date, Time) remains visible.

#### 1.3.3.3 Companions Ready Notification
*   A **push notification** is sent to the client indicating **"Your companions are ready"**.
*   **Timing rule:**
    *   If companion-companion matching (see [1.4.2](1.4_Matching_Flow.md)) is completed **before** **T−30 minutes**, the push notification is sent at **T−30 minutes**.
    *   If companion-companion matching completes **after** **T−30 minutes**, the push notification is sent **immediately** upon successful matching.

#### 1.3.3.4 Available Actions
*   **Cancel Booking** — Still available. Same flow as Part 1 (review page → cancellation finalized → redirect to **home screen**).
*   **Start Matching** — This button appears at **T−30 minutes** (enabled by the system). On tapping, the client is navigated to the **[Matching Page (1.4)](1.4_Matching_Flow.md)**. Companions transition from their "Ready & Waiting" screen to the client-companion matching page.
