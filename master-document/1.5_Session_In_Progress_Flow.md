# 1.5 Session In Progress Flow

## Scope
- The session-in-progress page displayed to client and companions once matching is complete and the session begins.
- Covers: session timer, SOS support, session extension, cancellation, session end, companion location monitoring, and companion-to-companion chat.

### 1.5.1 Client View

##### 1.5.1.1 Overview
*   This page appears automatically after the client-companion match is confirmed (see 1.4.1.7).
*   It remains active for the entire duration of the session.

##### 1.5.1.2 Visible Data
*   **Timer:** A running count-up timer showing **time elapsed** since session start.
*   Booking details: Venue, Date, Start Time.

##### 1.5.1.3 Actions

**A. SOS Button**
*   **1.5.1.3.1 Purpose:** Emergency support during the session.
*   **1.5.1.3.2 Action:** Triggers **in-app support** and sends an **alert to admin** simultaneously.

**B. Extend Session**
*   **1.5.1.3.3 Increment:** Fixed **+1 hour** extension. No other duration options.
*   **1.5.1.3.4 Limit:** Can only be extended **once** per booking.
*   **1.5.1.3.5 Approval:** No approval needed — instant from the client side. The companion is **not notified** — they just see the timer update on their screen.

**C. Cancel Session**
*   **1.5.1.3.6 Action:** Client taps "Cancel". A **confirmation modal** appears before the session is cancelled.
*   **1.5.1.3.7 On Confirm:** Session is ended immediately. Client is redirected to the **[Rating Page (1.6)](1.6_End_Session_Flow.md)**.

##### 1.5.1.4 Session End Notification
*   **1.5.1.4.1 Trigger:** At **15 minutes before the session end time**, a notification is sent to the client asking if they want to extend the session.
*   **1.5.1.4.2 Client Response:** Client can confirm **Yes** (extend by 1 hour) or **No** (session ends as scheduled).
*   **1.5.1.4.3 No Response:** If the client does not respond in-app, the companion will physically ask the client to confirm whether they want to extend.

##### 1.5.1.5 Session End
*   Once the session reaches the booked duration (2 hours, or 3 hours if extended), the session **auto-ends**.
*   **Status Transition:** Booking status changes from `ACTIVE` → `COMPLETED`.
*   Both client and companions are redirected to the **[Rating Page (1.6)](1.6_End_Session_Flow.md)**.

### 1.5.2 Companion View

##### 1.5.2.1 Overview
*   Same UI layout as the client view, but with different content.
*   This page appears for both Captain and Vice Captain after the client-companion match is confirmed.

##### 1.5.2.2 Visible Data
*   **Designation:** Displays the companion's role — **Captain** or **Vice Captain**.
*   **Timer:** A running count-up timer showing **time elapsed** since session start.

##### 1.5.2.3 Actions

**A. SOS Button**
*   **1.5.2.3.1 Purpose:** Emergency support during the session.
*   **1.5.2.3.2 Action:** Works the same as the client SOS — triggers **in-app support** and sends an **alert to admin** simultaneously.

**B. Message Box**
*   **1.5.2.3.3 Purpose:** In-app chat between **Captain and Vice Captain** only during the active session.
*   **1.5.2.3.4 Scope:** The client **cannot** see or participate in this chat.

##### 1.5.2.4 Location Monitoring
*   During the session, **both companions' GPS locations** are fetched at regular intervals.
*   The system checks that companions remain within **300 metres** of the client (as they are expected to be with the client at all times).
*   If a companion is detected too far from the client, the system sends an **alert to admin** and a **notification to the companion**.

##### 1.5.2.5 Limitations
*   Companions **cannot** extend or cancel the session. Only the client has those actions.
