## Scope
- Profile creation (signup) for both Client and Companion
- Login flow
- Home page for logged-in users
- Companion profile editing (nickname, languages, profile picture)

### 1.1 Onboarding & Profile Setup

#### 1.1.1 Signup (Client & Companion)

##### 1.1.1.1 Signup Form
*   Same form for both Client and Companion. Fields:
    *   **Name**
    *   **Nickname** (display name visible to others — never the real name)
    *   **Email**
    *   **Password**
    *   **Face Auth (Optional):** Option to enable **FaceID/TouchID** (device biometric) during password setup. If enabled, it can be used as a login shortcut.

##### 1.1.1.2 Email Verification
*   After signup, a **verification email** is sent to the user.
*   The user must verify their email before they can log in.

##### 1.1.1.3 API
*   **[POST] /auth/signup** — creates the user profile.

---

#### 1.1.2 Login

##### 1.1.2.1 Login Page
*   Simple login page with:
    *   **Email + Password** as the primary login method.
    *   **FaceID/TouchID** as a shortcut if the user enabled it during signup.

##### 1.1.2.2 API
*   **[POST] /auth/login** — authenticates the user and returns a session/token.

##### 1.1.2.3 Post-Login
*   On successful login, the user lands on the **Home Page**.
*   Only **logged-in users** can access the booking flow.

---

#### 1.1.3 Home Page

##### 1.1.3.1 Client Home Page
*   **No active booking:** Displays a **"Book Companion"** button. Tapping it navigates to the booking flow (see [1.2 Booking & Allocation Flow](1.2_Booking_And_Allocation_Flow.md)).
*   **Existing CONFIRMED booking:** Displays a **booking card** (Venue, Date, Time). Tapping it navigates to the **[Booking Confirmation Page (1.3)](1.3_Booking_Confirmation_Page.md)**. The "Book Companion" button is **disabled** (one booking at a time per 1.2.1.1).

##### 1.1.3.2 Companion Home Page
*   Displays the companion's **profile details**:
    *   **Name** (display name / nickname).
*   Displays **upcoming booking** card:
    *   **Before T−5h:** Shows only **Date & Time** (blocked slot, no venue or client details per 1.2.2.2).
    *   **After T−5h:** Shows **Date, Time, Venue, and Client Nickname** (detail reveal per 1.2.2.3).
*   **Active Toggle:**
    *   The companion can turn on an **"Active" toggle** at any time.
    *   Turning it ON signals the companion is ready for duty.
    *   The "Reached location?" button is **disabled** until the Active toggle is ON.
*   **"Reached location?" CTA:**
    *   Enabled only after the Active toggle is ON.
    *   On tap → **GPS check** is performed.
    *   **If at the venue:** Companion's arrival is confirmed.
        *   If the **other companion has also confirmed** → both are navigated to the **[Matching Page (1.4)](1.4_Matching_Flow.md)**.
        *   If the **other companion has NOT yet confirmed** → companion stays on the home page with a **"Waiting for partner…"** status message.
    *   **If NOT at the venue:** Error message is shown. Companion stays on the home page.
*   **Both companions** (Captain and Vice Captain) must independently activate their toggle and confirm location via "Reached location?" before the matching page loads for either of them.
*   **Performance Section** (bottom of the page):
    *   Displays the companion's **average star rating** (numeric format, e.g., 4.25 out of 5.00).

##### 1.1.3.3 Companion Profile Settings
*   Accessed via menu/navigation from the companion home page.
*   Allows companions to edit their profile:
    *   **Nickname:** Display name visible to clients and other companions (text input).
    *   **Languages:** Languages spoken (multi-select dropdown: English, Arabic).
    *   **Profile Picture:** Upload profile picture (JPEG or PNG, max 5MB). Displayed as sketch-style image.
*   Changes save immediately upon submission.
*   **API:**
    *   **[PATCH] /users/me** — updates nickname (Identity module).
    *   **[POST] /companion-profiles/upload-picture** — uploads profile picture.
    *   **[PATCH] /companion-profiles/me** — updates languages and profile picture URL.
