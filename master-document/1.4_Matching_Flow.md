# 1.4 Matching Flow

## Scope
- Client-Companion matching at the venue on the day of booking.
- Companion (Captain) - Companion (Vice Captain) matching before the client-companion phase.

### 1.4.1 Client-Companion Matching

##### 1.4.1.1 Overview
*   This is the phase where the assigned companion duo physically meets the client at the venue.
*   The client navigates to this page by tapping **"Start Matching"** on the Booking Confirmation Page (1.3.3.4) at **T−30 minutes**.
*   **Location access is mandatory** — the client must grant GPS permissions. There is no alternative.
*   **Companion GPS** is always on — companions are required to have GPS enabled at all times as part of their role.
*   The matching UI is the **same layout for both client and companions** — only the displayed information differs (similar to an Uber-style live map).

##### 1.4.1.2 Client Initiates Matching
*   The client's matching page shows companion live locations on the map **by default** (companions are already at/near the venue).
*   The page also shows a **"Start Matching"** button.
*   **Location Check:** The client must be **at the venue** (verified via GPS) before the "Start Matching" button becomes active.
*   Once the client taps "Start Matching", the companions begin seeing the **client's live location** on their matching page — making the location sharing **two-way**.

##### 1.4.1.3 Location Sharing
*   **Before "Start Matching":** One-way — client sees companion live locations; companions do **not** see the client.
*   **After "Start Matching":** Two-way — both sides see each other's live GPS on the map (updated at regular intervals).

##### 1.4.1.4 Matching Page — Client Side
*   **1.4.1.4.1 Trigger:** Client navigates here from the Booking Confirmation Page (1.3.3.4) by tapping **"Start Matching"** at **T−30 minutes**.
*   **1.4.1.4.2 Visible Data:**
    *   Companion live locations on the map (visible by default).
    *   Companion details (display names, ratings, languages, sketch profile pics) — same as the Part 2 reveal from 1.3.3.
    *   QR code for companions to scan (generated at T−30m).
    *   Backup PIN code for manual verification (generated at T−30m).
*   **1.4.1.4.3 Colour Screen Button:** Client can tap a button to display a **full-screen solid colour**. The colour is **unique and fixed per booking**. This helps companions visually spot the client at the venue.
*   **1.4.1.4.4 Actions:**
    *   **Start Matching** — active only when client is at the venue (GPS check).
    *   **End Booking** — client can end the booking if needed. This is the same as cancellation — booking is cancelled, client is redirected to home screen. No rating page.

##### 1.4.1.5 Matching Page — Companion Side
*   **1.4.1.5.1 Trigger:** After companion-companion matching is complete (1.4.2), companions transition from their **"Ready & Waiting" screen** to the client-companion matching page. This happens when the client taps "Start Matching" on the Booking Confirmation Page (1.3). Client's live location only appears **after the client taps "Start Matching"**.
*   **1.4.1.5.2 Visible Data:**
    *   Client Nickname.
    *   Live map with the client's location (after client starts matching).
    *   The booking colour to look for.
*   **1.4.1.5.3 Actions:**
    *   QR scanner to scan the client's QR code (Captain only).
    *   PIN entry field as backup (Captain only).
    *   **End Booking** button — companion can end the booking if needed. Same as cancellation — booking is cancelled, redirects to home screen. No rating page.

##### 1.4.1.6 Matching Mechanism
*   **1.4.1.6.1 Primary — QR Scan:** The **Captain** scans the client's QR code displayed on the client's matching page. Only the Captain needs to scan — the Vice Captain does not scan separately.
*   **1.4.1.6.2 Backup — PIN Entry:** If QR scan fails, the Captain types the **PIN code** (shown on the client's screen) into the companion app for manual verification.
*   **1.4.1.6.3 Colour Spotting:** The unique booking colour screen serves as a visual aid for companions to locate the client in a crowded venue. It is **not a verification mechanism** — QR or PIN is still required to confirm the match.

##### 1.4.1.7 Match Confirmation
*   Once the QR scan or PIN entry is successful, the session is officially started — the clock begins.
*   **Status Transition:** Booking status changes from `CONFIRMED` → `ACTIVE`.

### 1.4.2 Companion-Companion Matching (Captain — Vice Captain)

##### 1.4.2.1 Overview
*   Both companions must match with each other **before** the client-companion matching phase.
*   The matching page loads only after **both** companions have confirmed their location via "Reached location?" on the home page (see [1.1.3.2](1.1_Onboarding_And_Profile.md)).
*   This is a **simple QR-based matching** — no live map or GPS tracking between companions.

##### 1.4.2.2 Matching Page
*   Once both companions have confirmed location, they are both navigated to the com-com matching page.
*   **Captain's screen:** Displays a **QR code** (and a backup **PIN code**).
*   **Vice Captain's screen:** Displays a **QR scanner** (and a backup **PIN entry field**).

##### 1.4.2.3 Matching Mechanism
*   **1.4.2.3.1 Primary — QR Scan:** The **Vice Captain** scans the **Captain's QR code**.
*   **1.4.2.3.2 Backup — PIN Entry:** If QR scan fails, the Vice Captain types the Captain's **PIN code** into the app for manual verification.

##### 1.4.2.4 Match Confirmation
*   Once QR scan or PIN entry is successful:
    *   The companion duo is matched.
    *   A **push notification** is sent to the client indicating **"Your companions are ready"**.
        *   If companion-companion matching completes **before** **T−30 minutes**, the push notification is sent at **T−30 minutes**.
        *   If companion-companion matching completes **after** **T−30 minutes**, the push notification is sent **immediately** upon successful matching.
    *   Both companions see a **toast**: **"Match successful"**.
    *   Both companions are taken to a **"Ready & Waiting"** screen where they wait for the client-companion matching phase to begin.
*   **Ready & Waiting Screen — Visible Data:**
    *   Booking details: Venue, Date, Time.
    *   Status message: **"Waiting for client to start matching…"**
    *   Client Nickname.

##### 1.4.2.5 Failure Path
*   If the Captain and Vice Captain have **not matched by T−30m**, an **alert is sent to admin**. No automatic cancellation or other action at this time.
