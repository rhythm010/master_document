# Mobile Frontend V1 Milestones

This folder breaks the Version 1 mobile frontend roadmap into milestone-level implementation plans.

The main roadmap stays at `technical/frontend-companion/mobile-frontend-roadmap.md`.

Each milestone file should be used by future agents as the working brief for that phase. The goal is to keep every phase logic-first, testable, and clear about what is real functionality versus temporary placeholder UI.

## Milestone Files

- `milestone-00-real-device-placeholder-app.md`
- `milestone-01-core-app-foundation.md`
- `milestone-02-identity-flow-minimum-ui.md`
- `milestone-03-backend-driven-home-state.md`
- `milestone-04-booking-logic-flow.md`
- `milestone-05-native-capability-foundation.md`
- `milestone-06-matching-flow.md`
- `milestone-07-active-session-logic.md`
- `milestone-08-ratings-flow.md`
- `milestone-09-environment-release-hardening.md`
- `milestone-10-ui-design-polish.md`

## Agent Rules

- Build the milestone in order unless explicitly told otherwise.
- Keep early UI plain and functional.
- Do not implement final visual polish before Milestone 10.
- Register backend gaps in `technical/frontend-companion/backend-gap-register.md`.
- Use design references from `technical/frontend-companion/design-reference/v1/` when a milestone touches a known screen.
- Treat unsupported design controls as mock/static placeholders and document that clearly in the implementation.
