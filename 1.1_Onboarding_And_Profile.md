## Scope
- User Sign-Up
- Static Onboarding Pages explaining what Project Companion is
- Data schema definitions for Client and Companion personas

### 1.1 Onboarding & Profile Setup

#### Client Data Parameters (Required for Booking & Pre-Deployment Flows)

* **Identity & Contact**
    * Full Name (admin use only)
    * Nickname (displayed to companions)
    * Phone Number (for OTP, notifications)
    * Language Preferences (English/Arabic; controls UI only)
    * Payment Method (PCI-compliant token)
    * Push Notification Token/ID

* **Booking & Session**
    * Active Booking Status (enforces one active/future booking at a time)
    * Booking History (for audit/logging)
    * Location Permissions (GPS on/off, last known location)
    * Meeting Point (manual input or image upload; only for active booking)
    * QR Code/4-digit PIN (generated per booking, not stored in profile)

* **Session/Notifications**
    * Push Notification Token/ID (for real-time updates)

---

#### Companion Data Parameters (Required for Booking & Pre-Deployment Flows)

* **Identity & Profile**
    * Full Name
    * Phone Number (for notifications)
    * Language Skills (English/Arabic, others)
    * Physical Stats (as per onboarding)
    * Background Verification Status

* **Role & Assignment**
    * Role: Captain or Vice Captain (assigned by admin)
    * Shift Assignments (date, time, venue, duration)
    * Booking Assignments (linked to shift)
    * Calendar Availability (blocked slots, rest buffer)

* **Operational & Audit**
    * Device ID (per profile)
    * Device Audit Logs (per booking; for QR scan, matching, and logging)
    * GPS Permissions (must be always on)
    * Push Notification Token/ID

* **Session/Matching**
    * Current Duo Assignment (Captain/Vice, partner’s ID)
    * Matching Status (pending, confirmed, breach)
    * Last Known Location (for pre-arrival tracking)

* **Penalty/Violation**
    * Penalty Status (if any, for shift cancellation or breach)
    * Violation History (all past penalties/violations)
