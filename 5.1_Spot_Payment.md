## Scope
- End-to-end process of paying remotely for an item by companion on the request of the client.
- Covers initiation, item verification, strict 500 AED limit, and payment execution.

### 5.1 Spot Payment & Delegated Purchases
#### 5.1.1 Overview
This flow empowers the Companion to act as a purchasing agent for the Client, whether the Client is physically present ("Spot Payment") or remote within the venue ("Delegated Purchase"). All transactions are subject to strict financial limits and real-time Client approval.

#### 5.1.2 Functional (Companion)
##### 5.1.2.1 Initiation Modes
*   **Mode A: Spot Payment (Client Present)**
    *   **Context:** Client hands item to Companion at a store.
    *   **Action:** Companion takes item to counter, initiates "New Purchase".
*   **Mode B: Delegated Purchase (Remote)**
    *   **Context:** Client sends a request via **Command Dashboard (3.1)** (e.g., "Get me a latte from Starbucks").
    *   **Action:** Companion proceeds to venue.
    *   **Unavailability:** If the item/venue is unavailable, Companion **must** message Client via Dashboard for instruction. **No substitutions** without explicit approval.

##### 5.1.2.2 Item Details & Categorization
*   **5.1.2.2.1 Mandatory Form:** Before submission, Companion must input:
    *   **Category:** Dropdown selection (Fashion, F&B, Electronics, Accessories).
    *   **Brand/Store Name:** Text entry (e.g., "Zara").
    *   **Price:** Total amount.
*   **5.1.2.2.2 Limits:** Total Cart Value (including taxes/service charge) must be **< 500 AED**.
*   **5.1.2.2.3 Restrictions (Whitelist):**
    *   **Allowed:** Restaurants, Cafes, Retail Stores, Clothing/Fashion.
    *   **Explicitly Prohibited:** **Groceries**, Alcohol, Tobacco, Lottery, Gift Cards, Cash equivalents.
*   **5.1.2.2.4 Proof:** Photo of items with price tags or POS display is mandatory.

##### 5.1.2.3 Processing & Fulfillment
*   **5.1.2.3.1 Status PENDING:** Companion waits for Client approval at the counter.
*   **5.1.2.3.2 Offline Rule:** If connectivity is lost, Companion **must wait**. No offline transactions allowed.
*   **5.1.2.3.3 Timeout:** If Client does not respond after the allocated window (see 5.1.3.1), the request expires. Companion is instructed to "Abandon Purchase".
*   **5.1.2.3.4 Status AUTHORIZED:** 
    *   Companion pays using the prompt corporate card.
    *   **Action:** Companion **must** upload the physical receipt photo immediately.
*   **5.1.2.3.5 Returns:** Companion is **prohibited** from handling returns. Once bought, the item is the Client's responsibility.

#### 5.1.3 Functional (Client)
##### 5.1.3.1 Approval Request & Escalation
*   **5.1.3.1.1 Notification Level 1 (Initial):** Push notification: "Approve Payment: [Amount]".
*   **5.1.3.1.2 Notification Level 2 (Escalation):** If unanswered after **X minutes**, an automated **Bot Call** is triggered.
*   **5.1.3.1.3 Timeout:** If the call is unanswered/rejected, the request is auto-declined.

##### 5.1.3.2 Financial Logic
*   **5.1.3.2.1 Pre-Auth:** Approval triggers a **hold** on the Client's payment method.
*   **5.1.3.2.2 Failure:** If hold fails, Client is notified "Payment Failed". Companion sees "Do Not Pay".
*   **5.1.3.2.3 Settlement:** Hold is captured only after receipt upload.

#### 5.1.4 System Logic
*   **5.1.4.1 Hard Limit:** System rejects any cart total > 500 AED.
*   **5.1.4.2 Receipt History:** Digital receipts are instantly available in Client's transaction history.
